# U17-R8: docs / Doc consistency

## Summary

Documentation is largely accurate and well-maintained, but contains several stale references and discrepancies with the current implementation, mostly in ARCHITECTURE.md, AGENT_DESIGN.md, CONFIGURATION.md, and DOCUMENT_STORE.md. The most impactful issues involve incorrect model assignments in docs, a stale CLI interface description, and missing config fields.

## Notes

- The FLICK_INTEGRATION.md and FLICK_LIBRARY_MIGRATION.md docs are well-maintained and accurately reflect the current library-based integration. FLICK_INTEGRATION.md was updated post-migration.
- The FIX_LOOP_SPEC.md is accurate in its behavioral description -- leaf fix loop, branch fix loop, and scope circuit breaker all match the implementation in orchestrator.rs.
- Many discrepancies stem from documentation being written before implementation (design-first approach) and not being updated as implementation decisions diverged. The model assignment differences (findings 2-5) are particularly impactful since they affect cost and capability expectations.
- The services layer (DocumentStore, Research, Verification) consists almost entirely of stubs, which is inconsistent with DOCUMENT_STORE.md's detailed specification.
