# U4-R5: agent/prompts / Simplification

## Summary

The file follows a consistent pattern across eight prompt-builder functions but contains repeated boilerplate that could be consolidated with a small helper, and a recurring `.collect::<Vec<_>>().join()` idiom that can be simplified.

## Findings

| # | Severity | Finding | Location | Suggestion |
|---|----------|---------|----------|------------|
| 1 | minor | Every `build_*` function repeats the same 3-line `PromptPair { system_prompt, query }` construction after a `format!` call with identical `context = format_context(ctx)` interpolation. Eight instances of the same structural boilerplate. | `prompts.rs:79-106`, `108-134`, `136-163`, `165-195`, `197-229`, `231-257`, `259-288`, `290-334`, `336-356` | Extract a `PromptPair::new(system: &str, ctx: &TaskContext, suffix: &str) -> PromptPair` constructor (or similar) that calls `format_context` and appends the query suffix, eliminating the repeated pattern in each builder. |
| 2 | minor | The `collect::<Vec<_>>().join("\n")` idiom is repeated five times in `format_context` (ancestors, completed, pending) and once in `build_checkpoint` (disc_text). | `prompts.rs:27-28`, `48-49`, `58-59`, `274-275` | Use `itertools::join` or a small `join_lines` helper to avoid the intermediate `Vec` allocation each time. Alternatively, use `fold` or `intersperse` (nightly/itertools). |
| 3 | minor | `build_execute_leaf` and `build_fix_leaf` share nearly identical system prompts (same role intro, overlapping guideline bullets about reading files, minimal changes, verifying, and reporting discoveries). The fix variant adds one paragraph and tweaks wording. | `prompts.rs:109-122`, `166-179` | Extract shared guideline text into a constant or helper, composing the fix-specific additions on top. This reduces the chance of the two prompts drifting apart when guidelines are updated. |
| 4 | minor | `build_design_and_decompose` and `build_design_fix_subtasks` share overlapping system prompt content (subtask design guidelines, magnitude estimates, verification criteria, tool exploration). | `prompts.rs:137-151`, `198-212` | Factor shared decomposition guidelines into a constant and compose role-specific preambles around it. |
| 5 | note | The `"None".into()` default (returning `String`) appears three times in `format_context` for the empty-list branches of ancestors, completed, and pending. | `prompts.rs:21`, `31`, `52` | Minor, but a small `fn list_or_none(...)` helper would DRY this up and make the formatting of empty lists consistent by construction. |
| 6 | note | The tests follow a very uniform pattern: construct `test_context()`, call one builder, assert a few `contains` checks. Eight tests share this exact structure with only the builder call and assertions differing. | `prompts.rs:386-494` | Could use a parameterized test macro (e.g., `test_case` or a hand-rolled macro) to reduce repetition, though this is low priority since the tests are already clear. |

## Notes

Overall the code is well-structured and readable. The repetition is a natural consequence of having one function per prompt type, and each function is short enough that the boilerplate is not onerous. The suggestions above are incremental improvements rather than urgent refactors. The most impactful change would be finding #1 (a `PromptPair` constructor) combined with finding #3/#4 (shared guideline constants), which together would cut roughly 40-50 lines and reduce the risk of prompt guidelines drifting between related builders.
