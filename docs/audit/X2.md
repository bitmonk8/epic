The warnings cluster around three repeated patterns: `match` on `Option` instead of `let...else`, `Option::map_or_else` suggestions, and one unnecessary `Result` return type.

## Notes
- The build succeeded; all 12 clippy diagnostics are warnings, not errors.
- Findings 1-4 and 5-12 overlap: clippy fires `manual_let_else`, `single_match_else`, and `option_if_let_else` on the same four `match` blocks. Fixing with `let...else` (findings 1-4) would resolve all three lint groups at once, yielding 4 distinct code changes total.
- Finding 14 (`unnecessary_wraps`) is the only structurally distinct issue. The function currently returns `anyhow::Result<()>` but every code path returns `Ok(())` or early-returns without `?`. Changing the signature to `()` is a safe refactor.
- Nursery-level lints (`equatable_if_let`, `option_if_let_else`) are informational; they are not enabled by default even under `pedantic`.
