# U10-R4: config / Dead code & cruft

## Summary
The config module is clean with no TODO/FIXME markers, no stale allow annotations, and no ZeroClaw/YAML/subprocess remnants. However, two structs (`ProjectConfig` and `AgentConfig`) and their fields are defined but never accessed outside the config module itself, making them dead code carried forward from earlier design iterations.

## Findings
| # | Severity | Finding | Location | Suggestion |
|---|----------|---------|----------|------------|
| 1 | minor | `ProjectConfig` (fields `root`, `epic_dir`) is never imported or accessed outside `config/project.rs`. The `EpicConfig.project` field is deserialized but never read by any consumer. | `src/config/project.rs:25-30` (struct), `src/config/project.rs:9` (field) | Remove `ProjectConfig` and the `project` field from `EpicConfig`, or wire it into the orchestrator/init logic if the fields are intended for future use. |
| 2 | minor | `AgentConfig` (field `runtime`) is never imported or accessed outside `config/project.rs`. The `EpicConfig.agent` field is deserialized but never read by any consumer. | `src/config/project.rs:52-56` (struct), `src/config/project.rs:18` (field) | Remove `AgentConfig` and the `agent` field from `EpicConfig`, or wire `runtime` into the agent module where the Flick runtime is used. |
| 3 | note | Four `default_*` helper functions (`default_root`, `default_epic_dir`, `default_runtime`, and their `Default` impls) exist solely to support the two unused structs above. | `src/config/project.rs:68-69, 76, 79-83, 105-108` | Remove alongside the structs if they are dropped. |
| 4 | note | `mod.rs` is a one-line re-export (`pub mod project;`). The comment references `~/.config/epic/config.toml` but no global/user-level config loading exists yet. | `src/config/mod.rs:1` | Update the comment to reflect current scope (project-level config only), or remove the comment until user-level config is implemented. |

## Notes
- Grep searches for `TODO`, `FIXME`, `HACK`, `XXX`, `dead_code`, `allow(`, `zeroclaw`, `zclaw`, `serde_yaml`, `flick_path`, `work_dir`, `subprocess`, `spawn`, and `child process` returned zero matches in both files. The module is free of legacy markers and stale annotations.
- `Serialize` is derived on all config structs including the unused ones; this is correct because `init.rs` serializes the full `EpicConfig` to TOML, which requires all nested structs to implement `Serialize`.
