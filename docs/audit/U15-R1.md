# U15-R1: metrics / Correctness

## Summary
The metrics module (`src/metrics.rs` is a stub; the substantive code lives in `src/tui/metrics.rs`) is a small TUI widget that counts tasks by phase. No correctness bugs were found -- all `TaskPhase` variants are accounted for and there is no arithmetic, indexing, or fallible logic to go wrong.

## Findings
| # | Severity | Finding | Location | Suggestion |
|---|----------|---------|----------|------------|
| 1 | note | Subcategory counts are computed via four independent full iterations over the task map. While not a correctness issue, a single pass building a histogram would be cleaner and avoid redundant work on large task sets. | `src/tui/metrics.rs:26-50` | Iterate once, accumulate counts in a small array keyed by `TaskPhase` discriminant. |
| 2 | note | `src/metrics.rs` contains only a comment (`// Token usage and cost tracking.`) with no code. The module appears abandoned or not yet implemented; if it is declared in `mod` statements it adds dead weight. | `src/metrics.rs:1` | Remove the stub or implement the promised token/cost tracking. |

## Notes
All six `TaskPhase` variants (`Pending`, `Assessing`, `Executing`, `Verifying`, `Completed`, `Failed`) are explicitly matched, so the displayed subcategory counts always sum to `total`. No off-by-one, overflow, or missed error-path issues are present. The widget relies on ratatui's built-in clipping for undersized render areas, which is the idiomatic approach.
