# U17-R4: docs / Dead code & cruft

## Summary

Several documentation files contain stale references to the pre-migration subprocess/CLI integration pattern, removed CLI flags, and outdated configuration examples that contradict the implemented Flick library crate integration.

## Notes

- Findings 1-3 (ARCHITECTURE.md stale subprocess references) were previously identified in audit reports U2-R7 and U6-R7 but have not been addressed. They are re-reported here as they remain in the current document text.
- Finding 11 (STATUS.md historical entries) is rated as "note" because decision logs are expected to reflect the state at the time the decision was made. The subsequent migration entry documents the removal.
- The FLICK_INTEGRATION.md file correctly separates current state (library mode) from historical context (subprocess and ZeroClaw sections under "History"), making it a model for how ARCHITECTURE.md and DOCUMENT_STORE.md should be updated.
- AGENT_DESIGN.md, TASK_MODEL.md, VERIFICATION.md, TUI_DESIGN.md, FIX_LOOP_SPEC.md, and OPEN_QUESTIONS.md are clean -- no stale subprocess, ZeroClaw, serde_yaml, or removed-feature references found.
